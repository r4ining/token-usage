想要做一个统计 new-api 中某个或某些 key 使用的 token 的统计工具


功能如下：
1. 统计某个或某些 key（logs 表中的 token_name 字段） 在指定时间范围内的 token 使用量，需要可以按照天、周、月、所有时间等时间维度进行查看
2. 支持在页面配置模型的价格（输入、输出、缓存等），然后在输出的统计结果中，可以按照配置的价格计算出对应的费用（注意，配置时需要指定模型ID，上面 1 中统计出来的模型ID可能是与网上官方的不一致，用户配置的时候可以多个给某个模型配置一个模型别名，匹配到这个模型别名的也按照这个价格计算）
3. 在 1 的基础上，可以按照不同的模型进行分别统计，，具体支持情况如下：
    - 支持某个时间区间内汇总的每个模型的统计
    - 需要可以查看某段时间区间内每天的每个token的使用量
4. 支持导出结果为 Excel 表格


特别说明：
相关的逻辑可以参考 /Users/wangshihong/Git-repo/new-api（tag: v0.13.2）的代码，目前 new-api 前端“使用日志”页面会有每个请求的 token 使用量统计及费用的功能，可以参考他的这个查询/计算逻辑进行实现


数据库信息：
new-api 使用的数据库为 mysql，相关的地址、用户名、密码等信息需要可以通过环境变量配置

技术栈：
使用 go，前端自选，打包在一个镜像中，**需要在一个容器中运行**，不要分开前后端两个容器

部署方式：
通过 docker-compose 部署
